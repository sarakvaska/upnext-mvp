# Frontend/Backend Guidelines

## Frontend
- Use Tailwind CSS for styling
- Break components into small reusable pieces
- Favor simplicity and minimalism for creator workflows
- Create a responsive layout from day 1

## Backend
- Use Supabase as the default backend for speed
- Protect routes with Supabase Auth middleware
- Store files via Supabase Storage bucket and track metadata (version, section, etc.)
- Enable link-based sharing via signed URLs
